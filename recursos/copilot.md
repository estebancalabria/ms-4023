# Microsoft 365 Copilot Chat (Básico)

## Introducción

Microsoft 365 Copilot Chat (Básico) es el punto de partida para desarrollar fluidez en Inteligencia Artificial dentro del entorno laboral. Se trata de un asistente de IA empresarial incluido sin costo adicional en las suscripciones Microsoft 365, diseñado para ayudar a los usuarios a comenzar a trabajar con IA de forma segura, simple y preparada para organizaciones.

Copilot Chat (Básico) permite:
- practicar prompting,
- explorar ideas,
- generar contenido,
- investigar información,
- automatizar tareas repetitivas,
- aumentar productividad,
- desarrollar habilidades de trabajo con IA.

Todo esto ocurre inicialmente utilizando información basada en web, sin conectarse directamente a los datos internos de trabajo de la organización.

---

# ¿Qué es Microsoft 365 Copilot Chat (Básico)?

Microsoft 365 Copilot Chat (Básico) es un chat de Inteligencia Artificial empresarial que combina:
- modelos LLM modernos como GPT-5,
- grounding sobre información web,
- capacidades conversacionales,
- herramientas de productividad,
- seguridad empresarial integrada.

Funciona como un asistente de IA capaz de ayudar en tareas como:
- resúmenes,
- brainstorming,
- redacción,
- generación de ideas,
- codificación,
- análisis,
- investigación,
- búsqueda de información.

La experiencia está diseñada para ser intuitiva y basada en lenguaje natural.

---

# Objetivos de Copilot Chat (Básico)

El propósito principal de Copilot Chat (Básico) es:

# Grounding: Web vs Trabajo

## Modo Web

Copilot Chat (Básico) utiliza grounding basado en información pública web.

Esto permite:
- investigación,
- brainstorming,
- acceso a información actualizada,
- respuestas enriquecidas con datos públicos.

El modo Web NO accede a:
- archivos internos,
- correos,
- reuniones,
- chats organizacionales.

---

## Modo Trabajo (Premium)

Cuando se agrega una licencia de Microsoft 365 Copilot (Premium), aparece el modo Trabajo.

Este modo conecta Copilot con:
- archivos,
- correos electrónicos,
- reuniones,
- chats,
- datos organizacionales.

Esto permite:
- redactar documentos empresariales,
- resumir reuniones,
- consultar información interna,
- administrar tareas,
- trabajar directamente sobre contenido corporativo.

# Cómo usar Copilot Chat

## Paso 1: Escribir un prompt

Los usuarios interactúan mediante lenguaje natural.

Buenas prácticas:
- claridad,
- simplicidad,
- contexto,
- instrucciones específicas.

Ejemplos:
- pedir resúmenes,
- redactar contenido,
- generar ideas,
- analizar información.

---

## Paso 2: Enviar el mensaje

Puede enviarse mediante:
- botón Enviar,
- tecla Enter.

Copilot procesa la solicitud y genera una respuesta contextual.

---

## Paso 3: Refinar la conversación

Copilot permite:
- continuar conversaciones,
- pedir más detalles,
- resumir respuestas,
- generar refinamientos iterativos.

Ejemplos:
- “Escribe una respuesta más corta”
- “Dame más detalles”

---

# Transparencia y validación

Copilot muestra las fuentes utilizadas debajo de las respuestas.

Esto permite:
- validar información,
- revisar referencias,
- mantener transparencia.

Microsoft promueve un enfoque de:
- IA asistida,

# Enterprise Data Protection (EDP)

Uno de los pilares de Microsoft 365 Copilot es Enterprise Data Protection (EDP).

EDP es el conjunto de:
- controles de seguridad,
- privacidad,
- cumplimiento,
- gobernanza,

utilizados para proteger datos organizacionales.

---

# Cómo Copilot protege los datos

## Herencia de seguridad Microsoft 365

Copilot reutiliza:
- configuraciones existentes,
- compliance corporativo,
- políticas organizacionales.

---

## Cifrado y aislamiento de datos

Protección:
- en tránsito,
- en reposo.

Objetivo:
- evitar accesos no autorizados,
- asegurar confidencialidad.

---

## Privacidad por diseño

Los datos empresariales:
- NO se utilizan para entrenar modelos de IA.

Esto protege:
- propiedad intelectual,
- información sensible,
- contenido corporativo.

---

## Controles de acceso

Copilot respeta:
- permisos organizacionales,
- etiquetas de confidencialidad,
- políticas de retención,
- configuraciones de compliance.

---

## Cumplimiento regional

Los datos permanecen:
- dentro de la región geográfica correspondiente,
- alineados con compromisos regulatorios y de residencia de datos.
